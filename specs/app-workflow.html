Time Entry Workflow
พนักงานบันทึกชั่วโมงทำงานรายวัน/รายสัปดาห์ ลงใน Charge Code ที่ได้รับอนุญาต
Employee Login เข้าสู่ระบบ
Login ด้วย email/password หรือ Microsoft SSO ผ่าน Supabase Auth → เข้า Dashboard
ดู Dashboard สรุปสัปดาห์
แสดง progress bar ชั่วโมงที่บันทึก vs ที่ต้องบันทึก (เช่น 32h / 40h) พร้อม alert วันที่ยังไม่ครบ
Status Banner: Week Mar 16-22 → 32h/40h logged (80%)
Missing: Friday — 8h required
กดปุ่ม [Open Timesheet →] เพื่อไปหน้าบันทึก
เลือก Charge Code
เลือกจาก dropdown เฉพาะ charge codes ที่ได้รับอนุญาต (owner assign ให้) เพิ่ม/ลบแถวได้
แต่ละแถว = 1 charge code → กรอกชั่วโมงรายวัน (Mon-Fri)
แสดง tag [billable] หรือ [non-billable] ของแต่ละ code
เพิ่มคำอธิบายงานได้ (optional note icon)
กรอกชั่วโมงใน Grid
พิมพ์ชั่วโมงในแต่ละ cell → Tab ไปช่องถัดไป → Enter ลงแถวถัดไป
Validation:
• แต่ละวันต้องครบ 8 ชั่วโมงขั้นต่ำ (แสดง ✓ สีเขียว หรือ -8.0 สีแดง)
• วันหยุด/weekend → cell สีเทา disabled
• Auto-save ทุก 30 วินาที → status: DRAFT
Submit for Approval
กดปุ่ม [Submit →] เพื่อส่งให้ Line Manager อนุมัติ → Status เปลี่ยนเป็น "Submitted"
หลัง submit แล้วแก้ไขไม่ได้จนกว่าจะถูก reject
ระบบส่ง notification ให้ Line Manager ทันที
รอผลอนุมัติ
✓ Approved
Notification แจ้ง "อนุมัติแล้ว" → Timesheet ถูก lock
✗ Rejected
Notification แจ้ง "ถูก reject พร้อมเหตุผล" → กลับเป็น Draft → แก้ไขแล้ว submit ใหม่

Line Manager ขั้นที่ 1: Manager Approval
Manager เห็น pending timesheets ใน dashboard → เข้าหน้า Approvals
วิธีอนุมัติ:
• กด [Eye] ดูรายละเอียดแบบ inline expand → เห็น grid เต็ม
• เลือก checkbox หลายใบ → กด [Bulk Approve]
• หรือกด [Reject] → ใส่เหตุผล → timesheet กลับเป็น Draft

สิ่งที่ Manager ตรวจ: ชั่วโมงครบ? charge code ถูกต้อง? คำอธิบายสมเหตุสมผล?
CC Owner ขั้นที่ 2: Charge Code Owner Approval
หลัง Manager อนุมัติ → Charge Code Owner เห็นเฉพาะ entries ที่ charge ใน code ของตัวเอง
Toggle: [As Manager | As CC Owner] บน Approval page
CC Owner เห็นเฉพาะ entries ที่เกี่ยวข้องกับ codes ที่ตัวเองดูแล
อนุมัติ = ยืนยันว่างานนี้ควร charge ไปที่ code นี้จริง

Budget & Cost Tracking
คำนวณต้นทุนจริงจากชั่วโมงที่บันทึก เทียบกับ budget ที่ตั้งไว้

สูตรคำนวณต้นทุน

Actual Cost = Logged Hours × Cost Rate (by Job Grade)
Cost Rate Lookup: ดึง hourly rate จากตาราง CostRate ตาม job grade ของพนักงาน + วันที่ effective
ตัวอย่าง: Senior Engineer (Grade E4) → $85/hr → ทำงาน 8h → ต้นทุน $680/วัน

Budget Tracking — ตัวอย่าง

PRG-001 Digital Transformation $390K / $500K (78%)
PRJ-001 Web Portal Redesign $184K / $200K (92%) ⚠
PRJ-003 Data Migration $90K / $150K (60%)
Alerts ระบบแจ้งเตือนเมื่อ:
• Budget ถึง 80% → แจ้ง Warning (สีเหลือง)
• Budget ถึง 90% → แจ้ง Critical (สีแดง)
• Budget เกิน 100% → แจ้ง Overrun + ระบุ Activity/Task ที่เป็นต้นเหตุ
• Forecast: คาดการณ์ budget ณ สิ้นงวดจาก burn rate ปัจจุบัน

Automated & System Processes
Scheduled jobs ที่ทำงานอัตโนมัติ + Teams integration
Cron Auto-Cutoff: Lock Timesheets
ระบบ lock timesheets อัตโนมัติ วันที่ 15 และ สิ้นเดือน ของทุกเดือน
ใช้ @nestjs/schedule — cron job ทำงานตอน 00:00 ของวันที่ 15 และวันสุดท้ายของเดือน
Timesheets ที่ยังเป็น Draft/Submitted จะถูก lock โดยอัตโนมัติ → ไม่สามารถแก้ไขได้
Daily Reminder: Incomplete Timesheets
ทุกวัน 09:00 — ตรวจสอบพนักงานที่ยังไม่กรอกชั่วโมงครบ → ส่ง notification
ส่งผ่าน Teams notification + in-app bell icon
Manager ได้รับสรุปว่าทีมใครยังไม่ครบ
Daily Budget Alert Check
ตรวจสอบ charge codes ทุกตัวว่าเกิน threshold หรือไม่ (80%, 90%, 100%)
Weekly Insight Summary
ทุกวันจันทร์ 08:00 — สรุปสถานะ budget, chargeability, utilization ส่งให้ Program Owner และ Cost Center Owner
Teams Chatbot Integration
พนักงานพิมพ์ใน Teams chat เพื่อบันทึกเวลาหรือถามข้อมูล
ตัวอย่างคำสั่ง:
• "Logged 4h on PRJ-042 today" → ระบบ parse + บันทึกลง timesheet
• "What's my budget status for PRJ-042?" → ตอบ budget vs actual
• "Show my hours this week" → สรุปชั่วโมงที่บันทึกในสัปดาห์นี้
