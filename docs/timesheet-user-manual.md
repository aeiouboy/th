## 1. ภาพรวมระบบ 👤👔⚙️📊

ระบบ Timesheet เป็นเครื่องมือบันทึกเวลาทำงานและจัดสรรต้นทุนขององค์กร ใช้งานผ่านเว็บเบราว์เซอร์ได้ทุกอุปกรณ์

**ระบบนี้ทำอะไรได้:**

| ความสามารถ | รายละเอียด |
| --- | --- |
| **บันทึกเวลา** | กรอกชั่วโมงทำงานรายสัปดาห์ แยกตามรหัสงาน (Charge Code) |
| **ส่ง-อนุมัติ** | ส่งใบบันทึกเวลาให้หัวหน้าตรวจสอบและอนุมัติ |
| **ปฏิทินและวันลา** | ดูวันหยุด วันลา และขอลาพักร้อน |
| **ติดตามงบประมาณ** | เปรียบเทียบงบที่ตั้งไว้กับค่าใช้จ่ายจริง |
| **รายงาน** | วิเคราะห์ต้นทุน อัตรา chargeability แยกตามโปรแกรม/หน่วยงาน/บุคคล |
| **การแจ้งเตือน** | แจ้งเตือนกำหนดส่ง สถานะอนุมัติ และงบประมาณ |

**เมนูหลักทางซ้ายมือ (Sidebar):**

| ไอคอน | หน้า | ผู้ที่เห็น |
| --- | --- | --- |
| 🏠 Dashboard | หน้าหลัก | ทุกคน |
| 🕐 Time Entry | บันทึกเวลา | ทุกคน |
| 📅 Calendar | ปฏิทินและวันลา | ทุกคน |
| 🏷️ Charge Codes | รหัสงาน | หัวหน้า, ผู้ดูแล |
| ✅ Approvals | อนุมัติ | หัวหน้า, ผู้ดูแล |
| 📊 Reports | รายงาน | ผู้บริหาร, การเงิน |
| 💰 Budget | งบประมาณ | ผู้บริหาร, การเงิน |
| 👥 Admin (Users) | จัดการผู้ใช้ | ผู้ดูแลระบบ |
| 📅 Admin (Calendar) | จัดการปฏิทิน | ผู้ดูแลระบบ |
| ⚙️ Settings | ตั้งค่า | ทุกคน |

## 2. การเข้าใช้งาน 👤👔⚙️📊

### 2.1 เข้าสู่ระบบ (Login)

**ขั้นตอน:**

1. เปิดเว็บไซต์ระบบ Timesheet ขององค์กร
2. กรอก **อีเมล** ในช่องแรก (ตัวอย่าง: you@central.co.th)
3. กรอก **รหัสผ่าน** ในช่องที่สอง
   - กดปุ่ม **"Show"** ด้านขวาของช่องรหัสผ่าน เพื่อแสดงรหัสผ่านที่พิมพ์
4. กดปุ่มสีเขียว **"Sign In"**

**วิธีเข้าสู่ระบบอื่น:**

- กดปุ่ม **"Sign in with Microsoft"** (ปุ่มสีขาวมีโลโก้ Microsoft) เพื่อเข้าด้วยบัญชี Microsoft ขององค์กร
- ถ้าลืมรหัสผ่าน กดลิงก์ **"Forgot password?"** ด้านล่าง → ระบบจะส่งอีเมลให้ตั้งรหัสผ่านใหม่

### 2.2 ส่วนต่าง ๆ ของหน้าจอ

เมื่อเข้าสู่ระบบแล้ว จะเห็น:

- **แถบเมนูด้านซ้าย (Sidebar)** — ไอคอนสำหรับเปิดหน้าต่าง ๆ โดยมีโลโก้ **RIS** อยู่ด้านบนสุด (บางเมนูจะมองเห็นเฉพาะบางตำแหน่ง)
- **แถบด้านบน** — ปุ่มแฮมเบอร์เกอร์ (☰) + ชื่อหน้าที่เปิดอยู่
- **ปุ่มกระดิ่ง** — แจ้งเตือน (มุมขวาบน)
- **รูปโปรไฟล์** — มุมขวาบน กดเพื่อดูเมนูโปรไฟล์/ออกจากระบบ

### 2.3 ออกจากระบบ

กดที่ **รูปโปรไฟล์ (ตัวอักษรย่อ)** ที่มุมขวาบน → เลือก **"Log out"**

## 3. หน้าหลัก (Dashboard) 👤👔⚙️📊

> ทุกคนเห็นหน้านี้

![หน้าหลัก Dashboard](manual-screenshots/01-dashboard.png)

หน้าหลักแสดงภาพรวมการทำงานของคุณในสัปดาห์ปัจจุบัน:

### 3.1 ส่วนต้อนรับ (ด้านบน)

| สิ่งที่แสดง | ตัวอย่าง |
| --- | --- |
| **คำทักทาย + ชื่อ** | "Good evening, Tachongrak" |
| **ช่วงสัปดาห์** | "Week of Mar 30 - Apr 5, 2026" |
| **สถานะใบบันทึกเวลา** | ป้ายสี เช่น `Draft`, `Submitted`, `Approved`, `Rejected` |

### 3.2 แถบความคืบหน้า (Progress Bar)

- แสดง **ชั่วโมงที่บันทึกแล้ว / เป้าหมาย** เช่น "0.0h / 40h logged"
- ด้านล่างมีแถบแยกรายวัน **จ.–พฤ.** (และต่อไป) แสดงชั่วโมงแต่ละวัน

### 3.3 ปุ่มลัด (Quick Buttons)

| ปุ่ม | ทำอะไร |
| --- | --- |
| **My Codes** | ดูรหัสงานที่มอบหมายให้คุณ |
| **Approvals** (มีตัวเลขสีส้ม) | ดูใบบันทึกเวลาที่รอคุณอนุมัติ (สำหรับหัวหน้า) |

### 3.4 กล่องตัวเลข KPI (4 กล่อง)

| กล่อง | ความหมาย | ตัวอย่าง |
| --- | --- | --- |
| **Hours this period** | ชั่วโมงที่บันทึกในสัปดาห์นี้ / เป้าหมาย | 0 / 40 |
| **Chargeability** | อัตราการ charge งาน (เปอร์เซ็นต์) | 0% |
| **Pending approvals** | จำนวนใบที่รอคุณอนุมัติ | 1 |
| **Active charge codes** | จำนวนรหัสงานที่ใช้งานอยู่ | 25 |

แต่ละกล่องมีข้อมูลเปรียบเทียบ เช่น "−42h vs last period" หรือ "−100% vs prior"

### 3.5 การแจ้งเตือน (Notifications Panel)

![แผงการแจ้งเตือน](manual-screenshots/12-notifications-panel.png)

กดที่ **ไอคอนกระดิ่ง** มุมขวาบน → แผงแจ้งเตือนจะเลื่อนออกมาจากด้านขวา แสดงรายการแจ้งเตือนล่าสุด เช่น:

- สถานะ Pipeline (CI/CD)
- การอนุมัติที่รอดำเนินการ
- เตือนกำหนดส่ง

กด **"view all notifications →"** เพื่อดูทั้งหมด

## 4. การกรอกเวลา (Time Entry) 👤

> ทุกคนใช้หน้านี้

![บันทึกเวลาทำงาน](manual-screenshots/02-time-entry-grid.png)

หน้า Time Entry เป็น **ตารางรายสัปดาห์** (Weekly Grid) สำหรับกรอกชั่วโมงทำงาน

### 4.1 เลือกสัปดาห์

| กดตรงไหน | ทำอะไร |
| --- | --- |
| ปุ่ม **< (ลูกศรซ้าย)** ข้าง "Week of..." | ย้อนไปสัปดาห์ก่อนหน้า |
| ปุ่ม **> (ลูกศรขวา)** | ไปสัปดาห์ถัดไป |
| ช่อง **วันที่ dropdown** (เช่น "2026-03-30") | เลือกสัปดาห์ที่ต้องการโดยตรง |

ด้านบนแสดงข้อความเช่น **"Week of Mar 30 – 5, 2026"** พร้อมเดือนกำกับ (เช่น "March 2026")

### 4.2 โครงสร้างตาราง

ตารางประกอบด้วย:

| ส่วน | รายละเอียด |
| --- | --- |
| **หัวคอลัมน์** | Charge Code, Mon–Sat (พร้อมวันที่ เช่น "Mon 30", "Tue 31") รวมถึง Sun |
| **แถวรหัสงาน** | แต่ละแถวคือรหัสงาน 1 ตัว มีช่องกรอกชั่วโมงรายวัน |
| **Daily Total** | แถวสรุปยอดรวมรายวัน (ตัวหนา) |
| **Required** | แถวแสดงชั่วโมงที่ต้องทำ (วันทำงานปกติ = 8.00, วันหยุด = –) |
| **Variance** | ส่วนต่าง = Daily Total − Required ถ้าติดลบจะแสดงเป็น **ตัวแดง** (เช่น −8.0) |

### 4.3 เพิ่มรหัสงาน (Add Charge Code)

1. กดปุ่ม **"+ Add Charge Code"** ด้านล่างซ้ายของตาราง
2. จะแสดงรายการรหัสงานที่คุณมีสิทธิ์ใช้ → กดเลือกรหัสที่ต้องการ
3. รหัสงานจะปรากฏเป็นแถวใหม่ในตาราง

![กดปุ่ม Add Charge Code แล้ว dropdown แสดงรายการรหัสงาน](manual-screenshots/04-04-add-charge-code-dropdown.png)

![Charge Code ถูกเพิ่มเป็นแถวใหม่ในตาราง](manual-screenshots/04-05-charge-code-added.png)

> **ข้อสังเกต:** ถ้ายังไม่มีรหัสงานเลย จะเห็นข้อความ *"No charge codes added. Use the button below to add one."*

### 4.4 กรอกชั่วโมง

1. **กดที่ช่องว่าง** ในตาราง (จุดตัดระหว่างรหัสงาน กับ วัน)
2. พิมพ์จำนวนชั่วโมง (เช่น 8.00, 4.50, 2.00)
3. กด Tab หรือ Enter เพื่อย้ายไปช่องถัดไป
4. ระบบจะคำนวณ **Daily Total** และ **Variance** ให้อัตโนมัติ

![กรอก 8 ชม. วันจันทร์ — Daily Total แสดง 8.00](manual-screenshots/04-02-hours-entered.png)

**เป้าหมาย:** ทุกวันทำงาน (จ.–ศ.) ควรมียอดรวม **8.00 ชั่วโมง** — เมื่อครบ Variance จะเป็น 0.00

### 4.5 ลบรหัสงานออกจากตาราง

กดปุ่ม **× (กากบาท)** ที่ท้ายแถวรหัสงาน เพื่อลบรหัสงานนั้นออก (ข้อมูลชั่วโมงที่กรอกไว้จะหายไป)

### 4.6 คัดลอกจากสัปดาห์ก่อน

ถ้าใบบันทึกเวลาเป็น Draft และยังไม่มีรหัสงาน จะเห็นปุ่ม **"Copy from Last Period"** → กดเพื่อคัดลอกรหัสงานจากสัปดาห์ก่อนมาใช้ได้เลย ช่วยประหยัดเวลาไม่ต้องเพิ่มรหัสงานใหม่ทุกสัปดาห์

![กด Copy from Last Period — Charge Codes จากสัปดาห์ก่อนถูกคัดลอกมา](manual-screenshots/04-07-copy-from-previous.png)

### 4.7 ขอใช้รหัสงานใหม่

กดปุ่ม **"+ Request New CC"** เพื่อส่งคำขอใช้รหัสงานใหม่ไปยังผู้ดูแล (ใช้เมื่อรหัสงานที่ต้องการไม่อยู่ในรายการ)

### 4.8 การบันทึก

| วิธี | รายละเอียด |
| --- | --- |
| **Auto-save** | ระบบบันทึกให้อัตโนมัติทุก 30 วินาที (แสดงข้อความ "Auto-saves every 30s" ด้านล่าง) |
| **Save Draft** | กดปุ่ม **"Save Draft"** เพื่อบันทึกร่างทันที |

![กด Save Draft — ระบบบันทึกร่างสำเร็จ](manual-screenshots/04-03-save-draft.png)

### 4.9 OT และชั่วโมงพิเศษ

เมื่อกรอกรวมเกิน 8 ชม. ในวันเดียว ระบบจะแสดง **Variance สีแดง** (ค่าติดลบ) แต่ **ไม่บล็อก** — สามารถบันทึกได้ตามปกติ

![กรอก OT เกิน 8 ชม. — Variance แสดงสีแดง](manual-screenshots/04-06-overtime-variance.png)

#### 4.9.1 การบันทึก OT (Overtime)

การบันทึกชั่วโมงทำงานล่วงเวลาทำได้ดังนี้:

1. ในหน้า **Time Entry** กรอกชั่วโมงที่ **เกิน 8 ชั่วโมง** ในวันทำงานปกติ
   - ตัวอย่าง: กรอก 10.00 ในวันจันทร์ → Variance จะแสดง +2.0
2. สำหรับ OT ในวันหยุด: กรอกชั่วโมงในคอลัมน์ **Sat** หรือ **Sun**
   - Required จะแสดง "–" (ไม่มีเป้าหมาย) แต่คุณยังกรอกชั่วโมงได้

#### 4.9.2 รหัสงานสำหรับ OT

- ใช้รหัสงานเดียวกับงานปกติ — ระบบจะรวมชั่วโมงทั้งหมดในสัปดาห์นั้น
- หากองค์กรมีรหัสงาน OT แยกต่างหาก ให้เพิ่มรหัสงานนั้นผ่าน **"+ Add Charge Code"**

#### 4.9.3 วันลาและวันหยุด

- ชั่วโมงที่ลาจะถูกจัดการผ่านหน้า **Calendar** (ดูหัวข้อ 8)
- วันหยุดราชการแสดงในปฏิทินเป็น **สีชมพู/แดง** — ไม่ต้องกรอกชั่วโมงในวันเหล่านี้
- ถ้ามีรหัสงานสำหรับลาป่วย/ลากิจ ให้เพิ่มในตาราง Time Entry เหมือนรหัสงานปกติ

### 4.10 การส่งอนุมัติ (Submit for Approval)

#### 4.10.1 ตรวจสอบก่อนส่ง

ก่อนส่งใบบันทึกเวลา ตรวจสอบว่า:

- ทุกวันทำงาน (จ.–ศ.) มียอด **Daily Total = 8.00** ชั่วโมง
- แถว **Variance** แสดง 0.00 หรือ ✓ (ไม่มีตัวแดง)
- รหัสงานครบถ้วน ถูกต้อง

#### 4.10.2 ขั้นตอนการส่ง

1. กดปุ่ม **"Submit →"** สีเขียว มุมขวาล่างของตาราง
2. ถ้ายังมีวันที่ไม่ครบ 8 ชม. → ระบบจะแสดง **หน้าต่างเตือน** → กด **"OK, Got It"** แล้วกลับไปกรอกให้ครบ

![กรอกไม่ครบ 8 ชม. แล้วกด Submit — ระบบแสดง warning](manual-screenshots/05-01-submit-min-hours-warning.png)

3. เมื่อส่งสำเร็จ:
   - สถานะจะเปลี่ยนเป็น **"Submitted"**
   - หัวหน้าจะได้รับแจ้งเตือน
   - คุณจะไม่สามารถแก้ไขได้จนกว่าหัวหน้าจะตีกลับ (Reject)

![Submit สำเร็จ — badge เปลี่ยนเป็น Submitted](manual-screenshots/05-02-submitted-badge.png)

![Submitted แล้ว — fields เป็น read-only แก้ไขไม่ได้](manual-screenshots/05-03-submitted-readonly.png)

### 4.11 สถานะของใบบันทึกเวลา

| สถานะ | ป้ายสี | ความหมาย |
| --- | --- | --- |
| **Draft** | เทา | ร่าง — ยังแก้ไขได้ |
| **Submitted** | ฟ้า | ส่งแล้ว — รอหัวหน้าตรวจ |

## 5. ฟีเจอร์ AI (AI-Powered Features) 👤

ระบบ Timesheet มีฟีเจอร์ AI ช่วยเหลือการทำงาน ลดขั้นตอน และเพิ่มความแม่นยำในการบันทึกเวลา

![ปุ่ม Chat อยู่ที่มุมขวาล่างของทุกหน้า](manual-screenshots/ai-01-chat-button.png)

### 5.1 บันทึกเวลาด้วย AI (Record Time by AI)

AI สามารถช่วยกรอกชั่วโมงทำงานอัตโนมัติ โดยวิเคราะห์จากข้อมูลกิจกรรมของคุณ:

- **วิเคราะห์จากปฏิทิน** — ถ้าคุณมีนัดประชุมหรือ event ในปฏิทิน AI จะแนะนำชั่วโมงที่ควรกรอกในแต่ละวัน
- **เรียนรู้จาก pattern** — AI จดจำรูปแบบการทำงานของคุณ เช่น ถ้าปกติกรอก 8 ชม. ทุกวันจันทร์ถึงศุกร์ ระบบจะเสนอให้ auto-fill ได้เลย
- **ตรวจจับความผิดปกติ** — หาก AI พบว่าวันใดยังไม่มีการกรอกชั่วโมง จะแจ้งเตือนพร้อมคำแนะนำ เช่น "คุณยังไม่ได้กรอกวันพุธ — ปกติคุณบันทึก 8 ชม."

**วิธีใช้:**
1. เปิดหน้า **Time Entry** ตามปกติ
2. AI จะแสดงคำแนะนำชั่วโมงให้อัตโนมัติ (ถ้ามีข้อมูลเพียงพอ)
3. ตรวจสอบความถูกต้อง → กด **ยืนยัน** เพื่อใช้ค่าที่ AI แนะนำ หรือแก้ไขได้ตามต้องการ

> **หมายเหตุ:** AI เป็นผู้ช่วยแนะนำเท่านั้น — คุณยังคงตรวจสอบและยืนยันข้อมูลทุกครั้งก่อนส่ง

### 5.2 แนะนำ Charge Code อัตโนมัติ (Auto-Suggest Charge Codes)

เมื่อคุณเพิ่มรหัสงานหรือกรอกเวลา AI จะช่วยแนะนำ Charge Code ที่เหมาะสม:

- **แนะนำจากคำอธิบายงาน** — พิมพ์คำอธิบายสั้น ๆ เช่น "ทำงาน migration" → AI จะเสนอ Charge Code ที่ตรงกับงานนั้นโดยอิงจากประวัติการใช้งาน
- **แนะนำจากประวัติ** — Charge Code ที่คุณใช้บ่อยจะถูกเสนอก่อน ช่วยลดเวลาค้นหา
- **แนะนำจากทีม** — ถ้าเพื่อนร่วมทีมใช้ Charge Code ที่คุณยังไม่เคยใช้ AI อาจแนะนำให้ลอง

**วิธีใช้:**
1. กดปุ่ม **"+ Add Charge Code"** ในหน้า Time Entry
2. เริ่มพิมพ์ชื่อหรือคำอธิบายงาน → AI จะแสดงรายการแนะนำแบบ autocomplete
3. เลือกรหัสงานจากรายการ หรือพิมพ์รหัสเต็มด้วยตนเอง

### 5.3 AI Chatbot ช่วยเหลือ (AI Chatbot Assistance)

ระบบมี **Chat Widget** ในตัว สำหรับถามคำถามเกี่ยวกับ Timesheet ได้ทันทีโดยไม่ต้องออกจากหน้าที่ใช้งาน

**ความสามารถของ Chatbot:**

| สิ่งที่ถามได้ | ตัวอย่างคำถาม |
| --- | --- |
| **ดูสถานะ Timesheet** | "Timesheet สัปดาห์นี้ส่งหรือยัง?" |
| **ตรวจสอบ Chargeability** | "อัตรา chargeability ของฉันเท่าไหร่?" |
| **ดู Approval ค้าง** | "มีใบรออนุมัติกี่ใบ?" |
| **สอบถามงบประมาณ** | "โปรเจกต์ PRJ-001 งบเหลือเท่าไหร่?" |
| **ขอสรุป** | "สรุปชั่วโมงทำงานเดือนนี้ให้หน่อย" |

**วิธีใช้:**
1. กดไอคอน **💬 (Chat)** ที่มุมขวาล่างของหน้าจอ → แผง Chat จะเปิดขึ้น
2. พิมพ์คำถามเป็นภาษาธรรมชาติ (ไทยหรืออังกฤษ)
3. Chatbot จะตอบกลับพร้อม **ปุ่มลัด** (Suggested Actions) สำหรับคำถามที่เกี่ยวข้อง
4. กด Suggested Action เพื่อถามต่อได้ทันที

![เปิด Chat Panel — มี welcome message และปุ่มลัด](manual-screenshots/ai-02-chat-panel-open.png)

![ตัวอย่าง Chatbot ตอบรายการ Charge Code พร้อม Suggested Actions](manual-screenshots/ai-03-chatbot-response.png)

> **หมายเหตุ:** Chatbot แสดงข้อมูลเฉพาะที่คุณมีสิทธิ์เข้าถึงเท่านั้น — พนักงานเห็นเฉพาะข้อมูลของตัวเอง หัวหน้าเห็นข้อมูลทีม

## 6. การอนุมัติ (Approval Workflow) 👔

> สำหรับ **หัวหน้างาน** (Charge Manager) และ **ผู้ดูแลระบบ** (Admin)

![หน้าอนุมัติ](manual-screenshots/05-approvals.png)

### 6.1 ภาพรวมหน้า Approvals

หน้า Approvals มีส่วนหลัก:

- **หัวข้อ**: "Approvals — Review and approve submitted timesheets"
- **ตัวกรอง** (ด้านบน):
  - Dropdown เดือน เช่น `2026-03`
  - Dropdown สถานะ เช่น `all`
  - ช่องค้นหาชื่อ อีเมล หรือแผนก

### 6.2 แท็บต่าง ๆ

| แท็บ | คำอธิบาย |
| --- | --- |
| **Team Status** (ค่าเริ่มต้น) | ดูว่าลูกทีมแต่ละคนบันทึกเวลาครบหรือไม่ มีตัวเลขสีส้มบอกจำนวนทีม |
| **Pending Approvals** | ใบบันทึกเวลาที่รอคุณอนุมัติ (มีตัวเลขสีส้มบอกจำนวน) |
| **Vacations** | คำขอลาที่รอคุณอนุมัติ |
| **History** | ประวัติที่คุณเคยอนุมัติ/ตีกลับ |

### 6.3 แท็บ Team Status

แสดงสรุปสัปดาห์ปัจจุบัน:

| ข้อมูล | ตัวอย่าง |
| --- | --- |
| **ช่วงวันที่** | "Mar 30 — Apr 5" |
| **จำนวนวันทำงาน** | "5 working days" |
| **เป้าหมาย** | "40h target" |
| **สถานะทีม** | เช่น "⚠ 7 members with incomplete logging" (สีส้ม) |

ตารางด้านล่างแสดงรายชื่อพนักงาน:

| คอลัมน์ | ความหมาย |
| --- | --- |
| **Employee** | ชื่อ + อีเมล |
| **Status** | สถานะ เช่น Not Started, Draft, Submitted |
| **Hours Logged** | ชั่วโมงที่กรอก / เป้าหมาย เช่น "0 / 40h" |
| **Progress** | แถบเปอร์เซ็นต์ เช่น 0% |

### 6.4 อนุมัติทีละใบ

1. กดแท็บ **"Pending Approvals"**
2. กด **ไอคอนตา** ด้านขวาของแต่ละแถว เพื่อดูรายละเอียด
3. กดปุ่ม **✓ (อนุมัติ)** หรือ **✗ (ตีกลับ)**
   - ถ้าตีกลับ สามารถเขียนเหตุผลให้พนักงานรู้ได้

![Manager เห็น Pending Approvals — มีชื่อพนักงาน ชั่วโมง และปุ่ม Approve/Reject](manual-screenshots/06-01-manager-pending.png)

![กด Approve — ระบบแสดง "Timesheet approved"](manual-screenshots/06-02-manager-approved.png)

![กด Reject — ระบบเปิด dialog ให้กรอกเหตุผลการตีกลับ](manual-screenshots/06-03-rejection-dialog.png)

![พนักงานเห็นสถานะ Rejected พร้อมเหตุผลที่หัวหน้ากรอก](manual-screenshots/06-04-employee-rejected.png)

![พนักงานเห็นสถานะ Locked หลังอนุมัติเสร็จ — แก้ไขไม่ได้](manual-screenshots/06-05-employee-locked.png)

### 6.5 อนุมัติทีเดียวหลายใบ (Bulk Approve)

1. **ติ๊กช่องเลือก** หน้าแต่ละแถวที่ต้องการ
2. จะมี **แถบอนุมัติรวม** ขึ้นมาด้านล่าง
3. กดปุ่มอนุมัติรวม เพื่ออนุมัติทั้งหมดที่เลือก

### 6.6 อนุมัติวันลา

1. กดแท็บ **"Vacations"**
2. ดูชื่อพนักงาน วันที่ลา ประเภทการลา
3. กดปุ่ม **"Approve"** หรือ **"Reject"** ที่แต่ละรายการ

### 6.7 ลำดับขั้นตอนการอนุมัติ (Workflow)

```
พนักงานกรอกเวลา → กด Submit
        ↓
หัวหน้า (Charge Manager) ตรวจสอบ
        ↓
   ┌────────────┐
   │ Approve ✓  │ → สถานะ = Manager Approved → ส่งต่อ CC Owner
   └────────────┘
   ┌────────────┐
   │ Reject ✗   │ → สถานะ = Rejected → พนักงานแก้ไขแล้วส่งใหม่
   └────────────┘
        ↓ (กรณี Approve)
CC Owner (เจ้าของ Cost Center) ตรวจสอบ
        ↓
   ┌────────────┐
   │ Approve ✓  │ → สถานะ = CC Approved → ล็อกใบ
   └────────────┘
   ┌────────────┐
   │ Reject ✗   │ → สถานะ = Rejected → พนักงานแก้ไขแล้วส่งใหม่
   └────────────┘
```

| --- | --- |
| **Overview** | ภาพรวมทั้งหมด — กราฟ Budget vs Actual, Chargeability by Team, Activity Distribution |
| **By Program** | เลือกโปรแกรม → ดูรายละเอียดชั่วโมง ค่าใช้จ่าย แยกตามโปรเจกต์/กิจกรรม |
| **By Cost Center** | เลือกหน่วยงาน → ดูค่าใช้จ่ายแยกตามแผนก (เหมาะสำหรับฝ่ายการเงิน) |
| **By Person** | ค้นหาชื่อบุคคล → ดูชั่วโมง อัตรา charge งาน รหัสงานที่ใช้ |

### 9.3 ส่งออกข้อมูล (Export)

กดปุ่ม **"Export..."** มุมขวาบน → เลือก:
- **Export CSV** — ดาวน์โหลดเป็นตาราง (เปิดใน Excel ได้)
- **Export PDF** — สร้างรายงานแบบพิมพ์ได้

## 10. งบประมาณ (Budget) 📊

> สำหรับ **ผู้บริหาร, PMO, และการเงิน**

![งบประมาณ](manual-screenshots/07-budget.png)

### 10.1 ภาพรวมหน้า Budget

หน้า Budget Tracking แสดงหัวข้อ **"Budget Tracking"** พร้อมคำอธิบาย "Monitor charge code budgets, spending, and forecasts"

**ตัวกรอง:**

- Dropdown **"Programs"** — เลือกโปรแกรมหรือ "All (56)" เพื่อดูทั้งหมด

**กล่องสรุป KPI:**

| กล่อง | ตัวอย่าง | ความหมาย |
| --- | --- | --- |
| **Total budget** | ฿8.2M — "20 charge codes" | งบรวมทั้งหมด |
| **Actual spent** | ฿3.1M — "37% consumed" | ค่าใช้จ่ายจริง |
| **Forecast** | ฿11.1M — "Over by ฿2.8M" | คาดการณ์ค่าใช้จ่าย |
| **Status** | 7... — "3 over" | สถานะงบแยกตามสี |

### 10.2 ตาราง Budget by Charge Code

แสดงข้อความ "Click a row to see child-level breakdown" — กดแถวไหนก็ได้เพื่อขยายดูรายละเอียด

| คอลัมน์ | ความหมาย |
| --- | --- |
| **Charge Code** | ชื่อและรหัสงาน |
| **Budget** | งบที่ตั้งไว้ |
| **Actual** | ค่าใช้จ่ายจริง |
| **Usage** | แถบสี + เปอร์เซ็นต์การใช้งบ |
| **Forecast** | ค่าใช้จ่ายที่คาดการณ์ |
| **Variance** | ส่วนต่าง (+ = เหลืองบ, − = เกินงบ) |
| **Status** | ป้ายสีแสดงสถานะ |

### 10.3 ความหมายสี Status

| สี | ป้าย | ความหมาย |
| --- | --- | --- |
| เขียว | **On Track** | ปกติ ยังอยู่ในงบ |
| เหลือง | **Warning** | ใช้เกิน 80% — ควรระวัง |
| ส้ม | **Forecast Over** | คาดว่าจะเกินงบ |
| แดง | **Over Budget** | เกินงบแล้ว |

## 12. จัดการระบบ (Admin) ⚙️

> สำหรับ **ผู้ดูแลระบบ** (Admin) เท่านั้น

### 12.1 จัดการผู้ใช้ (Admin > Users)

![จัดการผู้ใช้](manual-screenshots/08-admin-users.png)

**กล่องสรุป:**
- **Total users** — จำนวนผู้ใช้ทั้งหมด (เช่น 7)
- **Admins** — จำนวนผู้ดูแลระบบ (เช่น 2)

**ตาราง User Management:**

| คอลัมน์ | ความหมาย |
| --- | --- |
| **Name** | ชื่อ-นามสกุล |
| **Email** | อีเมลองค์กร |
| **Role** | ตำแหน่งในระบบ เช่น `Charge Manager` (ป้ายสีเขียว), `Employee` |

| **Department** | แผนก เช่น Finance & Accounting, Merchandising, E-Commerce |

- ใช้ช่อง **"Search by name, email, or department..."** เพื่อค้นหา

### 12.2 จัดการปฏิทิน (Admin > Calendar)

![จัดการปฏิทิน](manual-screenshots/09-admin-calendar.png)

- แสดงปฏิทินทั้งปีเหมือนหน้า Calendar ของพนักงาน แต่มี dropdown เลือก **ประเทศ** (เช่น "TH") สำหรับกำหนดวันหยุดราชการ
- Breadcrumb แสดง "Admin > Calendar"
- สัญลักษณ์สี: Holiday (ชมพู), Weekend (เทา), Today (เขียว)
- ผู้ดูแลสามารถ **เพิ่ม/ลบวันหยุด** ได้จากหน้านี้

### 12.3 จัดการอัตราค่าใช้จ่าย (Admin > Rates)

![จัดการอัตรา](manual-screenshots/10-admin-rates.png)

**กล่องสรุป:**
- **Active rates** — จำนวนอัตราที่ใช้งาน (เช่น 13)
- **Avg hourly rate** — อัตราเฉลี่ยต่อชั่วโมง (เช่น ฿519.54)
- **Total rate records** — จำนวนรายการทั้งหมด

**Company Billing Rate:**
- แสดงอัตราเรียกเก็บของบริษัท เช่น **฿6,500.00/day** (฿812.50/hr)

**ตาราง Cost Rates:**

| คอลัมน์ | ความหมาย |
| --- | --- |
| **Job Grade** | ระดับงาน |
| **Hourly Rate** | อัตราต่อชั่วโมง เช่น ฿125.50 |
| **Effective From** | วันที่เริ่มใช้ เช่น Jan 1, 2026 |
| **Effective To** | วันที่สิ้นสุด (ถ้าไม่มี = ใช้ไปเรื่อย ๆ) |
| **Status** | สถานะ `Active` (สีเขียว) |

- ใช้ dropdown **"all"** เพื่อกรองตาม Job Grade
